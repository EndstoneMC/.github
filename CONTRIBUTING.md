# Contributing to EndstoneMC

Thanks for your interest in contributing! This guide applies to all repositories under the
EndstoneMC organization unless a repo has its own CONTRIBUTING.md.

## Getting Started

1. Fork the repository
2. Create a branch for your changes
3. Make your changes and test them
4. Open a pull request with a clear description

## Code Style

- Follow existing patterns in the codebase
- Run the linter if the repo has one configured (check the README)

## Contributor License Agreement (CLA)

By submitting a pull request, you agree that your contributions are your own original work
and that you grant EndstoneMC a perpetual, worldwide, non-exclusive, royalty-free license to
use, reproduce, modify, and distribute your contributions as part of the project under its
existing license.

First-time contributors will be asked to confirm this by commenting on their pull request.

## Reporting Issues

Use the repository's GitHub Issues for bugs and feature requests. Check for existing issues
before opening a new one.
